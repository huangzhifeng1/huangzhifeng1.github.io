hugo学习笔记

引言

HUGO 是一套模版静态化的系统，了解其目录结构有助于创建我们的网站系统

  • 官方文档:https://gohugo.io/documentation/

  • quick start:https://gohugo.io/getting-started/quick-start/

linux中安装hugo

去HUGO的官方网站中下载tar.gz文件至本机下载完成后用xftp将tar.gz文件上传至linux

相关指令:

tar -zxf hugo_0.107.0_Linux-64.bit.tar.gz `进行解压`
sudo cp ./hugo /usr/local/bin  `复制一份hugo程序到/usr/local/bin/`
sudo version `查看版本,判断是否安装成功`

hugo相关配置

hugo相关配置完成后，在相应的目录下建立站点

hugo new site fengge-blog

hugo相关主题: https://gohugo.io/getting-started/quick-start/

git clone + github地址

注意:需要将下载好的主题移动至themes目录

在虚拟机中用vim命令对config.toml文件内容进行修改！

vim config.toml

在本地启动hugo

hugo serve --bind "0.0.0.0" --buildDrafts

出现下图即为启动成功！

接下来，我们去浏览器测试一下，输入虚拟机的ip地址+1313端口号,看浏览器是否能显示页面！

在GitHub部署HUGO

首先我们需要在github中创建一个仓库，名字必须为github的用户名+github.io为后缀!


打开我们刚刚设置的仓库名字，仓库名字即是我们的访问网页! 打开后会出现404，是因为我们还没将hugo文件推送上github！

用hugo生成网页，托管到Github仓库中

hugo --theme=LoveIt --baseUrl="https://huangzhifeng1.github.io/" --buildDrafts

注意:theme后面为主题的名字！

执行完这条语句之后，会在当前目录下生成对应的public文件，最后我们需要将public推送至远程github！

cd public
git init
git add .
git commit -m "message"
git remote add origin + .....git `为我们远程推送的ssh`
git push -u origin master

在github上可以看到，文件已经成功推送成功啦！

现在我们可以进行访问我们刚刚部署的网站了，访问huangzhifeng1.github.io
